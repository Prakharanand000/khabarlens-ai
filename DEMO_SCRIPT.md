## PRE-RECORDING SETUP (Do this BEFORE hitting record)

1. Open localhost:5173 — wait for ALL stories to load fully
2. Make sure you have 10+ stories visible with images
3. Pre-open one article's Perspective modal → click "Why Polarized" 
   tab → run the analysis so it's cached. Then CLOSE the modal.
4. Paste a URL in the Article Analyzer → run it so results show
5. Open the AI Chat → ask "Which story is most polarized?" so it 
   has a response ready
6. Close everything back to the clean homepage
7. Make browser fullscreen (F11)
8. Clear browser URL bar clutter

## VOICEOVER TIPS

- Speak at a NORMAL pace — not fast, not slow
- Let the product speak for itself — don't over-explain
- When showing a feature, PAUSE for 1-2 seconds so judges can read
- Name each feature clearly: "Polarization Index", "Explainable 
  Polarization", "Article Analyzer", "Gen-Z Mode"
- Sound confident but natural — not salesy
- If something takes time to load, just say "as this loads..." 
  and narrate what it will show

## WHAT TO AVOID

- Don't show loading screens — pre-load everything
- Don't explain the code or architecture
- Don't say "um" or "so basically"
- Don't spend more than 15 seconds on any single feature
- Don't read text on screen word-for-word
- Don't apologize for anything ("sorry this is slow")
- Don't show the terminal/backend

Article Analyzer → AI Chat → Close.

Those 6 scenes cover the core value proposition.
